# 端到端工作流 详细说明

> 与 SKILL.md 配套，提供每一步的"具体做什么/不做什么/异常处理"。

## Step 1 hub（受理 + 澄清）

### 必做

1. 列出 `D:\项目文档\` 下所有子目录（含隐藏项目）
2. 如有多个项目 → 列出后让用户选择
3. 让用户确认目标文档类型（10 类之一）
4. 让用户说明意图（生成/更新/查询）

### 不做

- 不预设项目
- 不预设文档类型
- 不在用户没澄清时直接动手

### 异常

- 根目录非 `D:\项目文档\` → 提示用户确认
- 根目录无项目 → 提示用户提供项目根目录

## Step 2 query（资料抽取）

### 必做

1. 用 `DocumentLoader` 加载策划表（V1.0.xlsm），提取：
   - 里程碑日期
   - 评审计划
   - 损益分析
   - 风险登记册
2. 用 `glob` 列出项目已有同类 docx
3. 用 `project-doc-write/scripts/extract_docx_outline.py` 抽取 docx 章节作为格式范本

### 不做

- 不擅自打开非项目根目录的文件
- 不写临时脚本读文件（必须用 DocumentLoader）
- 不编造缺资料的内容

### 异常

- xlsm 加载失败（被锁）→ 提示关闭 Excel
- xlsm 没有"评审计划"子标签 → 模糊匹配
- 项目根目录无目标类型 docx → 提示用户上传范本
- PDF 文本 < 100 字符 → 视为扫描件，提示用户

## Step 3 outline（生成大纲）

### 必做

1. 加载 `references/outline_<类型>.md`
2. 按章节顺序输出（一级/二级标题）
3. 每章末标注"用途"和"必备小节"
4. 用户确认大纲

### 不做

- 不写正文
- 不擅自调整章节顺序
- 不擅自增删章节

### 异常

- reference 模板缺失 → 走 `outline_其他过程文档.md`
- 用户要求改章节 → 改后重出大纲，再确认

## Step 4 write（填充 + 决策）

### 必做

1. 按章节顺序逐章填充
2. 每章标注数据源（哪份文档/哪一节/哪一行）
3. 缺资料时主动询问用户（提供选项）
4. 生成"决策与意见"章节（带【框架】+【强度】+【数据源】）
5. 自检严禁虚构红线清单
6. 追加变更记录
7. 输出最终文档到项目目录
8. 输出中间稿到 `D:\项目文档\AIAssistive\output\<项目号>\`

### 不做

- 不编造任何数字/日期/人名
- 不写"应该/可能/或许"类无依据表述
- 不跳过变更记录

### 异常

- 用户拒绝提供缺资料 → 文档对应章节标"待补"
- 自检失败 → 必须重写对应章节
