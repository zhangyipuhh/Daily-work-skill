# 输出文件 命名规范

> 所有 project-doc 套件生成的文档，统一遵循本规范。

## 命名格式

### 项目目录（正式文档）

```
<项目根>/03_技术文档及评审/<子目录>/<类型名>.md
```

| 文档类型 | 子目录 | 文件名 |
|---|---|---|
| 售前方案 | 00_售前方案 | 售前方案.md |
| 需求说明书 | 02_需求分析 | 需求说明书.md |
| 概要设计说明书 | 03_概要设计 | 概要设计说明书.md |
| 详细设计说明书 | 03_概要设计 | 详细设计说明书.md |
| 实施方案 | 01_实施方案 | 实施方案.md |
| 测试方案 | 05_测试、验收 | 测试方案.md |
| 测试报告 | 05_测试、验收 | 测试报告.md |
| 验收报告 | 05_测试、验收 或 07_验收结题 | 验收报告.md |
| 实施部署方案 | 06_实施部署及培训 | 实施部署方案.md |
| 培训方案 | 06_实施部署及培训 | 培训方案.md |

### 变更记录（追加）

```
<项目根>/06_变更及暂停/变更记录.md
```

格式：

```markdown
| 日期 | 项目编号 | 变更类型 | 文档名 | 摘要 | 操作人（系统） |
|---|---|---|---|---|---|
| 2026-06-10 | 202410-C0008 | 新增 | 测试方案.md | 自动生成 V1.0 大纲与正文 | project-doc-skill |
```

### 中间稿（AIAssistive\output\）

```
D:\项目文档\AIAssistive\output\<项目号>/<类型名>_草稿.md
```

示例：

```
D:\项目文档\AIAssistive\output\202410-C0008/测试方案_草稿.md
```

> 遵循 AGENTS.md 规范"输出统一归集"。

```markdown
---

---
```
