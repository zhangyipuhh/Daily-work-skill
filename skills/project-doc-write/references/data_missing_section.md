# data_missing_section.md · 数据驱动章节无数据

> 配套 `intent-clarification/SKILL.md` 使用。
> owner skill：project-doc-write
> 场景分类：B. 数据完整性

## 触发条件

写正文时遇到**数据驱动章节**（如测试报告 / 测试总结 / 验收报告 / 实施总结），但项目资料中（策划表 / 需求 / 合同 / 邮件 / 周报）**无相关数据**。

**典型数据驱动章节**：
- 测试用例统计（计划/执行/通过/失败/通过率）
- 缺陷统计（按等级/按模块）
- 测试覆盖率
- 性能测试结果
- 验收测试结果
- 遗留问题清单

## 询问话术

```markdown
## 关于文档《[文档名]》的数据完整性询问

以下章节需要实际数据，但当前项目资料中无证据：

| 章节 | 所需数据 | 当前项目资料中是否有 |
|---|---|---|
| [章节号] [章节名] | [字段 1] / [字段 2] / ... | ❌ 无 |
| [章节号] [章节名] | [字段 1] / [字段 2] / ... | ❌ 无 |

请选择：
1. **提供数据**（请直接告诉我每个字段的具体值）
2. **保持"待补"状态**（按 §3.1 写法，不带"—"占位符）
3. **暂不生成**该文档（等数据齐了再写）
```

## 推荐写法（"待补"段）

```markdown
## 4.1 用例执行统计

**待补**：本章需项目组在测试执行后回填实际数据。

数据源建议：
- 测试用例库（按"测试阶段 × 计划/执行/通过/失败/阻塞/通过率"组织）
- 策划表中"K 点计划累计进度"列
- 实际测试执行记录

当前无数据源：策划表 / 需求 / 合同中均未提供测试执行数据。
```

## 不允许的写法

```markdown
## 4.1 用例执行统计

| 测试阶段 | 计划用例数 | 执行用例数 | 通过 | 失败 | 阻塞 | 通过率 |
| --- | --- | --- | --- | --- | --- | --- |
| 单元测试 | — | — | — | — | — | — |
```

❌ 表格全"—"占位是反模式（违反 write/references/word_格式范本_规则.md）。

## 询问结果分支

| 用户选择 | write 行为 |
|---|---|
| 提供数据 | 写入对应章节 |
| 保持"待补" | 用"待补"段格式写入，章节结构保留 |
| 暂不生成 | 退出 write 流程，汇报"数据不齐，待补 X 项" |
