# word 格式范本规则（写死版 · 基于项目实施方案.docx 沉淀）

> 本规则是从 `D:\项目文档\202410-C0008-泊头市关于开展地籍调查省级示范点建设项目\03_技术文档及评审\01_实施方案\实施方案V1.0.docx` 抽取的样式基准，已固化为以下硬规则。
> 写死原因：保持 write skill 的"高内聚低耦合"——不依赖 docx-skill/word-skill 等第三方 skill。
> 重抽/重写：可手动修改本文件，或删本文件后重跑 `scripts/extract_word_style.py` + 人工校对。

---

## 1. 封面页（强制 · 第 1 页）

| 元素 | 规则 |
|---|---|
| 项目名称 | 居中，**黑体二号**（22pt），段前 6cm 段后 1cm |
| 项目编号 | 居中，**黑体三号**（16pt），段前 1cm 段后 0.5cm |
| 文档名 | 居中，**黑体二号**（22pt），段前 1.5cm 段后 0.5cm |
| 版本 + 日期 | 居中，**宋体小四**（12pt），段前 0.5cm 段后 0.5cm |

> **不写"评审稿/草稿/正式版"等状态标签**（除非用户在策划表/需求中明确要求）
> **不写"编制/审核/批准"角色表**（除非用户在策划表中明确指定了具体姓名）
> 封面页**不计入页码**

---

## 2. 目录页（强制 · 第 2 页）

- 自动从 1-3 级标题生成
- 占 1 页
- **双层目录**：
  - **markdown 静态层**（在 .md 中以 `## 目录` + 链接列表形式存在，模型/读者可直接跳转）
  - **docx 自动层**（转 .docx 时由 docx-skill 用 Word 自动目录覆盖，更新页码）
- 目录页**不计入页码**

---

## 3. 正文样式

| 元素 | 规则 |
|---|---|
| 一级标题（H1 / `## 1 引言`） | **黑体小三**（15pt），段前 24pt 段后 18pt，单倍行距 |
| 二级标题（H2 / `### 1.1 范围`） | **黑体四号**（14pt），段前 18pt 段后 12pt，单倍行距 |
| 三级标题（H3 / `#### 1.1.1`） | **黑体小四**（12pt），段前 12pt 段后 6pt，1.5 倍行距 |
| 正文 | **宋体小四**（12pt），行间距 1.5 倍，首行缩进 2 字符 |
| 表格内文字 | **宋体小四**（12pt），行间距 1.25 倍 |
| 代码/路径 | Consolas 五号（10.5pt） |
| 引用/注释 | 楷体小四（12pt），左侧缩进 2 字符 |
| 列表 | 正文样式 + 项目符号/编号 |

---

## 4. 表格样式

| 元素 | 规则 |
|---|---|
| 表头 | 加粗 + 居中，**黑体小四**（12pt） |
| 单元格边框 | 0.5pt 黑色实线 |
| 列宽 | 自适应（按内容最长单元格） |
| 表头底纹 | 浅灰（如 #D9D9D9） |

---

## 5. 页眉页脚

| 元素 | 规则 |
|---|---|
| 页眉左 | 项目编号 + 项目名（宋体小五 9pt） |
| 页眉右 | 文档名（宋体小五 9pt） |
| 页脚居中 | 第 X 页 / 共 Y 页（宋体小五 9pt） |
| 页码起点 | 目录页之后的正文首页（封面 + 目录不计入） |
| 页眉横线 | 0.5pt 黑色实线 |

## 7. 不写的内容（硬约束）

> 这些**不允许出现在生成的文档中**，除非用户明确要求：

- ❌ 状态标签：`评审稿` / `草稿` / `正式版` / `待评审` / `终稿` / `V0.X`
- ❌ 角色签名表：`编制` / `审核` / `批准`（除非策划表中有具体姓名）
- ❌ 元占位段：`> 评审稿阶段尚未填写具体数据` / `> 待 XX 阶段补充`
- ❌ "—" 占位符（用 `**待补**：[字段名]` 替代）
- ❌ 空话引用：`参见项目模板库中《XX》模板` / `详见附件`
- ❌ 项目专有工具：禅道 / Jira / Git / 钉钉 / 飞书（除非项目资料中明确提到）
- ❌ 项目资料中未出现的具体人名/日期/数字（**严禁虚构**）

## 8. markdown 必备结构

每个 .md 文档**必须**含以下结构（write Step 4.1 强制）：

```markdown
# <文档名>

<封面信息：项目名/编号/版本/日期>

---

## 目录

- [1 引言](#1-引言)
  - [1.1 编写目的](#11-编写目的)
  - [1.2 范围](#12-范围)
- [2 <章节名>](#2-章节名)
- [...]

---

## 1 引言

### 1.1 编写目的
...
```

> 目录必须手写（markdown 不支持自动 TOC），但每条链接锚点要正确。

---

## 9. 章节编号规范

- 一级：`## 1 引言`（阿拉伯数字）
- 二级：`### 1.1 编写目的`
- 三级：`#### 1.1.1 子节`
- 不使用"第一章"等中文数字
- 不使用"一、"等顿号编号

---

## 10. markdown 语法映射（写给 docx-skill 参考）

| markdown 元素 | docx 元素 | 样式 |
|---|---|---|
| `# H1` | Title 样式 | 黑体二号居中（封面用） |
| `## H2` | Heading 1 | 黑体小三 + 段前后间距 |
| `### H3` | Heading 2 | 黑体四号 + 段前后间距 |
| `#### H4` | Heading 3 | 黑体小四 + 段前后间距 |
| `\| 表格 \|` | Table | 表头加粗 + 0.5pt 边框 + 浅灰底纹 |
| `> 引用` | Quote | 楷体小四 + 左缩进 |
| ` ```代码``` ` | Code | Consolas 五号 |
| `- 列表` | List Paragraph | 项目符号 |
| `1. 列表` | List Paragraph | 编号 |
| `**粗体**` | Run(bold=True) | — |
| `*斜体*` | Run(italic=True) | — |
| `[文字](URL)` | Hyperlink | 蓝色下划线 |

## 11. 文档状态约定

write 默认不写"评审稿/草稿"等状态。如用户在策划表/需求中明确要求写状态，按以下措辞：

| 用户原话 | 文档中写 |
|---|---|
| "出 V1.0 初稿" | `文档状态：初稿` |
| "出评审稿" | `文档状态：评审稿` |
| "正式版" | `文档状态：正式版` |
| 没说 | **不写状态字段**（保持沉默） |

---

## 12. 角色签名表约定

write 默认不写"编制/审核/批准"角色表。如策划表/需求中明确指定了具体姓名，按策划表填写：

| 角色 | 来源 |
|---|---|
| 编制 | 策划表中"项目负责人"或"编制人"字段 |
| 审核 | 策划表中"审核人"或"项目经理"字段 |
| 批准 | 策划表中"批准人"或"项目发起人"字段 |

策划表中**没有** → 文档中**不写**该角色。
策划表中**有** → 文档中按策划表填写。
