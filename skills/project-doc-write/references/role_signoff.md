# role_signoff.md · 角色签名表

> 配套 `intent-clarification/SKILL.md` 使用。
> owner skill：project-doc-write
> 场景分类：D. 文档属性

## 触发条件

write 准备在文档末尾添加"编制 / 审核 / 批准"角色签名表时。

## 询问话术

```markdown
## 关于文档《[文档名]》的角色签名表

项目资料中是否有具体姓名映射到以下角色？

| 角色 | 策划表/合同/需求中是否有具体姓名？ |
|---|---|
| 编制 | [如"项目负责人：张三" / ❌ 无] |
| 审核 | [如"项目经理：李四" / ❌ 无] |
| 批准 | [如"项目发起人：王五" / ❌ 无] |

请选择：
1. **按策划表填写**（如果上表有具体姓名）
2. **保持沉默**（项目资料中无具体姓名 → 文档不写角色表）
3. **我现在指定**（直接告诉我每个角色对应的人）
```

## 推荐行为

| 资料情况 | 行为 |
|---|---|
| 策划表/需求中有"项目负责人"等字段 | 按字段填写角色表 |
| 资料中无 | **不写角色表**（保持沉默） |
| 用户主动指定 | 按用户指定填写 |

## 严禁行为

```markdown
❌ | 编制 | 张三 |     ← 策划表里没"张三"
❌ | 角色 | 姓名 |     ← 资料中无任何映射
❌ | 编制 | 待定 |
```

## 推荐格式（资料中有具体姓名时）

```markdown
| 角色 | 姓名 |
|---|---|
| 编制 | [策划表 §项目负责人] |
| 审核 | [策划表 §项目经理] |
| 批准 | [策划表 §项目发起人] |
```
