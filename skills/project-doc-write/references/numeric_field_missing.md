# numeric_field_missing.md · 数值字段缺失

> 配套 `intent-clarification/SKILL.md` 使用。
> owner skill：project-doc-write
> 场景分类：B. 数据完整性（子项：数值字段）

## 触发条件

写正文时遇到**具体数值字段**缺失：
- 工时（如"总工时 200 人天"）
- 成本（如"项目预算 100 万"）
- 工期（如"工期 6 个月"）
- 数量（如"用例数 150"）
- 百分比 / 比率（如"通过率 95%"）
- 时间点（如"2025-12-10 完成"）

## 询问话术

```markdown
## 关于 [字段名] 的数据完整性询问

章节 [章节号] 提到 [字段名]：
> "..."

当前项目资料（策划表/合同/需求/周报）中：
- ❌ 无该字段具体值
- ✅ 有相关线索（如下游字段）：[如有]

请选择：
1. **提供具体值**：[请直接给数字]
2. **提供范围**：[如 "约 200 人天" / "100-150 万"]
3. **引用现有文件**：[如 "按合同 §3.1 第 5 行"]
4. **保持"待补"**：用 **待补** 标记，等数据齐全后回填
```

## 推荐写法（"待补"格式）

```markdown
- 总工时：**待补**（策划表未给具体值）
- 工期：6 个月（合同 §3.1 第 5 行）
- 成本：100-150 万（用户口述）
```

或：

```markdown
- 总工时：约 200 人天（**待补**：精确值需项目组提供）
```

## 严禁写法

```markdown
- 总工时：—
- 总工时：TBD
- 总工时：待定
```

❌ "—" / "TBD" / "待定" 单独使用是反模式（无具体说明）。

## 询问结果分支

| 用户选择 | write 行为 |
|---|---|
| 提供具体值 | 直接填入 |
| 提供范围 | 写入范围 + 注明"约"或"待补：精确值..." |
| 引用现有文件 | 写入"按 [文件] §X 第 Y 行" |
| 保持"待补" | 用"**待补**：XXX"格式 |
