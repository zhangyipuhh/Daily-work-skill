# 实施方案 格式范本 提取说明

> 在 `project-doc-write` 阶段，套件需要从项目已有的"实施方案.docx"中抽取章节结构作为格式范本。
> 本说明规定提取路径、回退顺序与产出格式。

## 提取路径

```
<项目根>/03_技术文档及评审/01_实施方案/*.docx
```

## 回退顺序

```
01_实施方案/*.docx
  └─ 不存在 → 02_需求分析/*.docx
              └─ 不存在 → 03_概要设计/*.docx
                          └─ 不存在 → 询问用户上传格式范本
```

## 提取脚本

```bash
python scripts/extract_docx_outline.py \
  --docx "<docx_path>" \
  --output "<output_md_path>"
```

> 脚本走本 skill **自带的** `scripts/DocumentLoader.py`（路径为 `../project-doc-write/scripts/DocumentLoader.py`），通过 `scripts/loader/WordLoader.py` 处理 docx；
> heading 抽取直接用 `python-docx`（保留层级信息，DocumentLoader 的 WordLoader 不保留 heading 层级）。

## 产出格式

```markdown
# 格式范本：实施方案

## 一级章节 1
### 1.1 二级章节
### 1.2 二级章节
## 一级章节 2
### 2.1 二级章节
...
```

> 仅抽取章节标题（Heading 1/2/3），不抽取正文。

## 用途

- 作为 `project-doc-outline` 章节标题的微调参考
- 作为 `project-doc-write` 正文排版的样式参考（标题层级/编号风格）

## 异常处理

| 异常 | 处理 |
|---|---|
| docx 文件被 Word 锁定 | 提示关闭 Word 后重试 |
| docx 中无 Heading | 提示用户手动确认章节结构 |
| 多个 docx 同名 | 取修改日期最新者 |
