# 决策建议生成规则

> 在 `project-doc-write` 生成"决策与意见"章节时使用本规则。

## 输入

- 项目策划表（xlsm）
- 合同/需求/方案
- 周报/里程碑报告
- 当前文档的撰写上下文

## 输出

按 `references/decision_advisory_template.md` 格式的"决策与意见"章节。

## 决策类别（按 PMP 知识领域）

| 类别 | 触发条件 | 数据源 |
|---|---|---|
| 范围决策 | 合同范围 vs 实际交付 | 合同 + 需求 + 周报 |
| 进度决策 | 计划 vs 实际偏差 > 5% | 策划表《里程碑》 + 周报 |
| 成本决策 | 计划成本 vs 实际成本 | 策划表《损益分析》 + 周报 |
| 质量决策 | 测试不通过/缺陷密度超阈 | 测试报告 + 策划表《质量目标》 |
| 风险决策 | 风险登记册 vs 实际触发 | 策划表《风险登记册》 + 周报 |
| 资源决策 | 人月投入与计划不一致 | 策划表《损益分析》 + 周报 |
| 沟通决策 | 重要干系人未参会 | 沟通计划 + 会议纪要 |
| 采购决策 | 采购到货延迟 | 采购合同 + 实施报告 |
| 变更决策 | 客户提出新需求 | 客户邮件 + 变更控制记录 |
| 干系人决策 | 干系人冲突/满意度低 | 干系人登记册 + 沟通记录 |

## 决策强度

| 强度 | 含义 | 触发 |
|---|---|---|
| 强建议 | 应当执行 | 多源数据一致支持 |
| 弱建议 | 可考虑 | 单一数据源支持 |
| 提示 | 需要关注 | 数据趋势出现异常 |
| 警告 | 必须响应 | 偏差超容差 |

## 强制输出格式

每条决策与意见**首行**：

```
【框架：{PMP|PRINCE2|系统分析师} · {框架层|落地层|实务层}】
【类别：{范围|进度|成本|质量|风险|资源|沟通|采购|变更|干系人}】
【强度：{强建议|弱建议|提示|警告}】
【数据源：策划表《X》子标签 + 文档路径 N】
【决策点】：<具体问题>
【依据】：<具体数据>
【建议】：<具体行动>
【风险】：<不执行的后果>
```

## 严禁

- 凭空假设
- 无数据源
- 无框架标签
- 一句话敷衍
- 用"行业最佳实践"代替本项目数据
