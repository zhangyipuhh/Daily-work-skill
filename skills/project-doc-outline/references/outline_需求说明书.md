# 需求说明书 大纲模板

> 适用：明确"做什么"
> 评审：内部评审 + 客户评审（外部）
> 来源参考：项目根目录/03_技术文档及评审/02_需求分析/ 需求说明书.docx

## 章节结构

1. **引言**
   - 1.1 编写目的
   - 1.2 项目背景
   - 1.3 术语与缩略语
   - 1.4 参考资料
2. **项目概述**
   - 2.1 项目目标
   - 2.2 用户角色
   - 2.3 运行环境
   - 2.4 假设与约束
3. **功能需求**
   - 3.1 功能划分（按模块）
   - 3.2 功能描述（每个功能：描述/输入/输出/处理逻辑/前置条件/后置条件）
   - 3.3 业务流程
4. **非功能需求**
   - 4.1 性能需求
   - 4.2 安全性需求
   - 4.3 可靠性需求
   - 4.4 易用性需求
   - 4.5 可维护性需求
   - 4.6 可移植性需求
5. **数据需求**
   - 5.1 静态数据
   - 5.2 动态数据
   - 5.3 数据字典
6. **接口需求**
   - 6.1 用户接口
   - 6.2 外部接口（硬件/软件/通信）
   - 6.3 内部接口
7. **验收标准**
8. **附录**
   - 8.1 用例图
   - 8.2 业务流程图
   - 8.3 数据流图
