# 策划表子标签说明模板

> 策划表（V1.0.xlsm）是项目**指导性文件**，所有交付物的提交时点、评审时点、责任人、损益估算均在此。
> 本模板定义套件读取策划表时的子标签结构与字段提取规范。

## 典型子标签（按行业实践 / GB/T 8564 / ISO 21500）

| 子标签 | 用途 | 关键字段 |
|---|---|---|
| **封面** | 项目编号、名称、版本、密级、责任人 | 项目编号、项目名称、V1.0、责任人、日期 |
| **总体计划概述** | 阶段划分、关键里程碑 | 阶段、起始日期、结束日期、交付物 |
| **项目里程碑** | 里程碑节点 | 里程碑名称、计划日期、状态、关联交付物 |
| **WBS / 工作分解** | 任务清单 | WBS 编号、任务名、责任人、起止日期、工期 |
| **编码估算** | 行数/功能点估算 | 模块、代码行数估算、复用率 |
| **汇总损益分析** | 人月/成本估算 | 阶段、人月、成本、收入 |
| **风险登记册** | 风险项 | 风险描述、概率、影响、应对策略、责任人 |
| **假设与约束** | 项目假设 | 假设描述、影响范围 |
| **质量目标** | 质量指标 | 指标名、目标值、度量方法 |
| **沟通计划** | 沟通机制 | 沟通对象、频率、方式 |
| **变更控制** | 变更流程 | 变更分级、审批权限 |
| **评审计划** | 评审安排 | 文档名、提交日期、评审日期、评审方式、评审人 |
| **甘特图 / 时间线** | 可视化时间线 | 任务条、里程碑 |
| **责任人矩阵（RACI）** | 责任分配 | 任务、负责人、执行、咨询、知会 |
| **配置管理计划** | 基线/版本规则 | 基线名称、版本号、纳入基线日期 |

## 评审计划子标签（最高频使用）

> 用户问"什么时候交/什么时候评审"时，**直接读此表**。

| 字段 | 含义 |
|---|---|
| 交付物名称 | 测试方案 / 实施方案 / 需求说明书 / ... |
| 提交日期 | 内部完成日 |
| 评审日期 | 评审会议日 |
| 评审方式 | 会议评审 / 邮件评审 / 同行评审 |
| 评审人 | 评审人名单（可能多个） |
| 评审结论 | 通过 / 有条件通过 / 不通过（评审后填写） |

## 损益分析子标签（用于决策建议）

- 阶段人月 vs 实际人月 → 进度偏差
- 计划成本 vs 实际成本 → 成本偏差
- 风险登记册 vs 实际风险触发 → 风险命中

> 套件生成"决策与意见"时，**必须**从这些子标签的实际值与基线值的差中提炼，不允许凭空假设。

## 字段提取注意事项

1. xlsm 中可能含 VBA 宏公式，套件使用 openpyxl 的 `data_only=True` 读取**计算后**的值
2. 隐藏 sheet 默认**不**包含（`include_hidden=False`），除非用户明确要求
3. 表格结构以纯文本渲染（`\t` 分隔），便于后续切片检索
4. 子标签名称可能因项目而异，套件**模糊匹配**上述关键词（如"里程碑"/"Milestone"）
