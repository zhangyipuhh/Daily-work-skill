| 02 | `02_需求分析` | **需求说明书** | 评审通过后归档 |
| 03 | `03_概要设计` | **概要设计说明书** | 评审通过后归档 |
| 04 | `04_UI设计` | UI 原型/规范 | 评审通过后归档 |

| 06 | `06_实施部署及培训` | **实施部署方案、培训方案** | 实施前评审 |
| 07 | `07_项目过程文档` | 周报、会议纪要、检查单 | 持续归档 |
| 08 | `08_产品清单` | 产品清单 | 验收时核对 |

> **关键提示**：
> - 01/02/03/05/06 五个子目录是**软件工程项目技术文档**的核心目录
> - 项目过程文档放 07（不要混进 03 各子目录）
> - 格式范本默认从 01 实施方案取；为空时回退到 02 需求分析 → 03 概要设计

## 路径识别规则（glob）

- 策划表（V1.0.xlsm）：`<项目根>/01_项目策划/*策划表*V*.xlsm` → 取 V 后数字最大者
- 策划评审通知：`<项目根>/01_项目策划/*策划*.eml` 或 `*.pdf`
- 实施方案范本：`<项目根>/03_技术文档及评审/01_实施方案/*.docx`（取第一个；为空时回退）
- 变更记录：`<项目根>/06_变更及暂停/变更记录.md`（不存在则创建）
