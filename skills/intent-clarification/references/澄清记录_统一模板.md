# 澄清记录统一模板

> 配套 `intent-clarification/SKILL.md` 使用。

## 文件位置

**主日志**：`<用户工作根>/.project/<project_id>/project_log.md`
**澄清记录**：`<用户工作根>/.project/<project_id>/clarification_log.md`

## 模板（首次创建时）

### project_log.md

```markdown
# Project Log · 202410-C0008

> 项目：泊头市关于开展地籍调查省级示范点建设项目
> 项目根：<项目根目录绝对路径>
> 初始化时间：2026-06-11 12:00:00
> 初始化 skill：project-doc-overview

## 操作记录

| 时间戳 | skill | 操作类型 | 修改内容 | 证据/路径 |
|---|---|---|---|---|
| 2026-06-11 12:00 | project-doc-overview | 初始化 | 创建 .project 目录与本文件 | .project/202410-C0008/ |
```

### clarification_log.md

```markdown
# Clarification Log · 202410-C0008

> 配套 project_log.md 使用。
> 每次模型向用户询问并得到回答后追加一行。

| # | 时间戳 | 维度 | 子项 | 问题摘要 | 用户回答 | 来源 | 调用的 skill |
|---|---|---|---|---|---|---|---|
| 0 | 2026-06-11 12:00 | (init) | (init) | 初始化空日志 | (init) | (init) | project-doc-overview |
```

## 来源字段约定

| 来源标识 | 含义 |
|---|---|
| `用户口述` | 用户直接回答（无项目资料依据） |
| `用户确认` | 项目资料已有，用户确认采纳 |
| `策划表 §X 第 Y 行` | 从策划表具体位置引用 |
| `合同 §X 第 Y 条` | 从合同具体位置引用 |
| `需求 §X 第 Y 行` | 从需求说明书具体位置引用 |
| `澄清日志 #N` | 引用之前澄清记录 |

## 追加操作

通过 `manage_project_log.py`：

```bash
# 追加澄清记录
python scripts/manage_project_log.py append-clarification \
    --project-id 202410-C0008 \
    --work-root "D:\项目文档" \
    --dimension "C.environment" \
    --item "tech_hardware" \
    --question "是否按已有信息写？" \
    --answer "是" \
    --source "策划表 §2 第 3 行 + 用户确认" \
    --asked-by "project-doc-outline"

# 追加主日志（skill 流程结束）
python scripts/manage_project_log.py append-operation \
    --project-id 202410-C0008 \
    --work-root "D:\项目文档" \
    --skill "project-doc-write" \
    --op-type "新增" \
    --content "测试方案.md" \
    --evidence "项目根/03_技术文档及评审/05_.../测试方案.md"
```

## 路径外部性原则

**关键**：所有 `.md` 文件**不在 skill 内**。原因：
- skill 是只读资产，不应被运行时修改
- 跨项目隔离（不同项目不同日志）
- 用户工作目录可备份/版本管理（可选）
