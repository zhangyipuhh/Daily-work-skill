# 澄清协议统一规范

> 配套 `intent-clarification/SKILL.md` 使用。

## 询问流程

```
模型/下游 skill 需要问用户
  ↓
1. 识别维度（intent/data/environment/document_attr）
  ↓
2. 读 <用户工作根>/.project/<project_id>/clarification_log.md
   ├─ 已有同子项记录 → 展示 + 问"确认/更新"
   └─ 无记录 → 进入步骤 3
  ↓
3. 调 query skill 的 read_doc.py 扫项目资料
   ├─ 命中 → 引用文件路径+行号+原文摘要 + 问"按已有信息写？/补充？"
   └─ 未命中 → 直接问
  ↓
4. 用户回答"待定"？
   ├─ 是 → 再次询问"停止流程 / 提供详细信息"
   └─ 否 → 进入步骤 5
  ↓
5. 调 manage_project_log.py append-clarification 追加记录
  ↓
6. 把答案返回给调用方 skill
  ↓
7. 流程结束后调 manage_project_log.py append-operation 写主日志
```

## 询问话术模板

每个维度的具体话术见各 skill 的 reference：

- 流程澄清：`<用户工作根>/.project/<project_id>/references/intent.md`（如已建）
- 实际位置：`project-doc-query/references/intent.md`
- 数据完整性：`project-doc-write/references/data_missing_section.md` 等
- 环境/技术/合规：`project-doc-outline/references/tech_*.md`（10 个）
- 文档属性：`project-doc-write/references/role_signoff.md` / `doc_status.md`

## 记录格式

### clarification_log.md

```markdown
| # | 时间戳 | 维度 | 子项 | 问题摘要 | 用户回答 | 来源 | 调用的 skill |
|---|---|---|---|---|---|---|---|
| 1 | 2026-06-11 12:34 | A.intent | project_root | 项目根目录？ | D:\项目文档\202410-C0008-... | 用户口述 | project-doc-hub |
| 2 | 2026-06-11 12:35 | C.environment | tech_hardware | 是否根据已有信息写？ | 是 | 策划表 §2 第 3 行 | project-doc-outline |
| 3 | 2026-06-11 12:36 | C.environment | tech_database | 数据库选型？ | 达梦 8 | 用户口述 | project-doc-outline |
```

### project_log.md

```markdown
| 时间戳 | skill | 操作类型 | 修改内容 | 证据/路径 |
|---|---|---|---|---|
| 2026-06-11 12:00 | project-doc-overview | 初始化 | 创建 .project 目录 | .project/202410-C0008/ |
| 2026-06-11 12:30 | project-doc-outline | 新增 | 测试方案.md | 项目根/03_技术文档及评审/05_.../测试方案.md |
| 2026-06-11 12:35 | project-doc-write | 更新 | 测试报告.md（净化） | 同上 |
| 2026-06-11 12:40 | intent-clarification | 澄清 | tech_hardware 已确认 | .project/202410-C0008/clarification_log.md#3 |
| 2026-06-11 12:50 | project-doc-workflow | 追加 | 变更记录.md | 项目根/06_变更及暂停/变更记录.md |
```

## 路径解析

- `<用户工作根>` = 项目根目录的父目录
  - 例：项目根 `D:\项目文档\202410-C0008-...` → 工作根 `D:\项目文档\`
- `<项目号>` = 解析项目根目录名前 2 段
  - 例：`202410-C0008-...` → `202410-C0008`
- `.project 目录` = `<用户工作根>/.project/`
- 项目子目录 = `<用户工作根>/.project/<项目号>/`

由 `dump_paths.py` 自动生成这些路径变量（PowerShell `. (ps1)` 加载）。
