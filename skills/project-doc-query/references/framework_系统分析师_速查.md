# 软考高级·系统分析师 速查（1 页）

> **国考体系** · 软件工程 + 信息系统 + 信息化 · 软件项目实际工程问题与架构 · 实务层

## 5 大知识模块

| # | 模块 | 关键能力 |
|---|---|---|
| 1 | 系统规划 | 可行性分析、立项、总体规划 |
| 2 | 需求分析 | 业务需求、用户需求、功能/非功能需求、需求建模 |
| 3 | 系统设计 | 架构设计、详细设计、数据库设计、接口设计 |
| 4 | 测试与维护 | 测试策略、测试用例、缺陷管理、系统维护 |
| 5 | 信息化 | 信息系统集成、信息安全、电子商务、政务信息化 |

## 与项目文档的对应

| 模块 | 对应项目文档 |
|---|---|
| 系统规划 | 售前方案、立项报告、可行性研究 |
| 需求分析 | 需求说明书、需求规格说明 |
| 系统设计 | 概要设计、详细设计、数据库设计、接口设计、UI 设计 |
| 测试与维护 | 测试方案、测试报告、验收报告、维护手册 |
| 信息化 | 实施部署方案、运维方案、培训方案 |

## 软件工程核心能力（与项目文档对接）

- **需求分析能力**：DFD、IDEF0、用例图、类图、状态图
- **系统设计能力**：架构模式（MVC/微服务/分层）、数据库范式、接口规范
- **测试能力**：黑盒/白盒/灰盒、单元/集成/系统/验收测试
- **维护能力**：改正性/适应性/完善性/预防性维护
- **信息化能力**：信息集成、安全体系、标准规范

## 套件使用建议

- 用户问"软件工程中的实际问题"（性能/架构/数据库/接口）→ 用系统分析师 实务层
- 用户问"需求怎么写才完整" → 需求分析模块
- 用户问"测试覆盖率/测试策略" → 测试与维护模块
- 用户问"实施运维/培训" → 信息化模块
