## 文档类型识别关键字（用户表述 → 标准类型）

| 用户可能说的 | 套件映射 |
|---|---|
| 售前、立项、可行性 | 售前方案 |
| 需求、规格、SRS | 需求说明书 |
| 概要、HLD、总体设计 | 概要设计说明书 |
| 详细、LLD、模块设计 | 详细设计说明书 |
| 实施、落地、上线 | 实施方案 |
| 测试方案、测试计划 | 测试方案 |
| 测试报告、测试总结 | 测试报告 |
| 验收、签收、终验 | 验收报告 |
| 部署、运维、迁移 | 实施部署方案 |
| 培训、演练 | 培训方案 |

## 文档位置动态识别规则

- 标准编号的子目录 → 按上表
- 编号与项目不完全一致 → 提示用户确认位置
- 文档不存在 → 询问用户：
  - 要"生成新文档"还是"在已有草稿基础上改"？
  - 是否提供格式范本？
